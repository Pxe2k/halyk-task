Для транзакций используем Serializable самый строгий, но самый надежный уровень изоляции транзакций который помогает нам избегать таких проблем как:
- Грязное чтение (Dirty Read)
- Неповторяемое чтение (Non-repeatable Read)
- Фантомное чтение (Phantom Read)

Из минусов работает медленнее, но для бронирования как мне кажется это важно.
Вызывает чаще других уровней изоляций Rollback, но у нас есть Retry.
